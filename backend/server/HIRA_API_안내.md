# HIRA 병원정보 Open API 500 에러 시 확인 사항

## 1. 인증키 종류
- 공공데이터포털에서 **일반인증키(비인코딩)** 값을 사용하세요.
- (인코딩) 키를 쓰면 애플리케이션에서 한 번 더 인코딩되어 500/인증 오류가 날 수 있습니다.

## 2. 활용신청 API
- **건강보험심사평가원_병원정보서비스** 중 **getHospBasisList1(병원기본목록)** 에 대해 활용신청을 했는지 확인하세요.
- 다른 API(예: 의료기관별상세정보서비스) 키는 이 서비스에 사용할 수 없습니다.

## 3. 승인 및 대기 시간
- 활용신청 후 **약 30분** 뒤부터 사용 가능하다는 안내가 있습니다. 신청 직후라면 잠시 후 다시 시도해 보세요.

## 4. 브라우저에서 직접 호출 테스트
아래 URL에서 `YOUR_KEY`를 **비인코딩 인증키**로 바꿔서 브라우저 주소창에 넣어 보세요.

```
http://apis.data.go.kr/B551182/hospInfoService1/getHospBasisList1?ServiceKey=YOUR_KEY&pageNo=1&numOfRows=10&sidoCd=110000&_type=json
```

- **200 + JSON** → 인증/파라미터는 정상. 애플리케이션 쪽만 점검하면 됩니다.
- **500 또는 에러 메시지** → 키/활용신청/서비스 상태를 포털에서 다시 확인해야 합니다.

## 5. 트래픽
- 개발계정은 **일 1,000건** 제한이 있습니다. 초과 시에도 오류가 발생할 수 있습니다.
